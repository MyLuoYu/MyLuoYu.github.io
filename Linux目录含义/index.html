Linux目录含义

发表于 2022-11-25 更新于 2022-12-01 分类于 IT知识

在Linux根目录（/）下有各种目录，其存放的文件或目录都有有一套标准依据。了解他们能够方便我们对文件进行配置更改。

环境为：Centos7

Linux目录配置的依据—FHS

​ 因为利用Linux来开发产品或发行版本的社区/公司与个人实在太多了，如果每个人都用自己的想法来配置文件放置的目录，那么将可能造成很多管理上的困扰。于是就有所谓的Filesystem Hierarchy Standard （FHS）
根据FHS的标准文件指出，他们的主要目的是希望让使用者可以了解到已安装软件通常放置于那个目录下，所以他们希望独立的软件开发商、操作系统制作者、以及想要维护系统的使用者，都能够遵循FHS的标准。也就是说，FHS的重点在于规范每个特定的目录下应该要放置什么样子的数据而已。

根(/)目录的内容和意义

​ 根目录是整个系统最重要的一个目录，所有的目录都是由根目录衍生出来的。因为根目录是这么的重要，所以在FHS的要求方面，他希望根目录不要放在非常大的分区内， 因为越大的分区你会放入越多的数据，如此一来根目录所在分区就可能会有较多发生错误的机会。

根下各个目录含义

在Linux的世界里，一切皆文件。

路径 含义
/bin binary的缩写，存放 Linux自带的操作命令，如find, grep, cat, chmod, chown, date等常用命令。
/boot 放置开机会使用到的文件，包括Linux核心文件以及开机菜单与开机所需配置文件等。
/dev 在Linux系统上，任何设备与周边设备都是以文件的型态存在于这个目录当中的。 你只要通过存取这个目录下面的某个文件就等于存取某个设备如cpu、disk等。
/etc 存放系统的配置档，系统主要的配置文件几乎都放置在这个目录内，例如人员的帐号密码档、 各种服务的启始档等等。
/lib或/lib64 来存放函数库。
/media Linux系统会自动识别一些设备，如U盘、光驱等。识别后，Linux会把识别的设备挂载到该目录下。
/mnt 存放用户临时在外部挂载的文件。
/opt 存放第三方软件的目录，以前的Linux系统中，还是习惯放置在/usr/local目录下。
/usr usr是Unix Software Resource的缩写， 也就是“Unix操作系统软件资源”所放置的目录。
/run 系统开机后所产生的各项信息。
/sbin 放在/sbin下面的为开机过程中所需要的，里面包括了开机、修复、还原系统所需要的指令。 至于某些服务器软件程序，一般则放置到/usr/sbin/当中。指令包括：fdisk, fsck, ifconfig, mkfs等等。
/srv service的缩写，存放一些网络服务启动之后，这些服务所需要取用的数据目录。
/tmp 用来存放 一些临时文件。
/var 存放不断扩充的东西，包括各种日志文件。
/proc 他放置的数据都是在内存当中， 例如系统核心、行程信息（process）、外部设备的状态及网络状态等等。
/sys 跟/proc非常类似，主要也是记录核心与系统硬件信息较相关的信息。 包括目前已载入的核心模块与核心侦测到的硬件设备信息等等。
/root 系统管理员的用户主目录。
/home 存放 root之外的普通用户。
/lost+found 这个目录是使用标准的ext2/ext3/ext4文件系统格式才会产生的一个目录，目的在于当文件系统发生错误时， 将一些遗失的片段放置到这个目录下。不过如果使用的是 xfs 文件系统的话，就不会存在这个目录了！
